# visual-check — 視覚検証スキル

## 手順

1. `pnpm build` → `pnpm preview --port 4173 --strictPort` をバックグラウンド起動
2. `CHROMIUM_PATH=/opt/pw-browsers/chromium node scripts/shots.mjs <出力dir>`
   (ライト/ダーク両方 × 全タブが撮れる)
3. **全スクリーンショットを Read で自分の目で見る** (ライトとダーク両方。片方だけで済ませない)
4. `pnpm test` — `src/a11y/contrast.test.ts` がパレットの WCAG 比率を強制する
5. 問題があれば修正して 1 に戻る。通ったら PR にスクショを添付/送付

## チェックリスト (WCAG 2.2 のうちこのアプリで有用なもの)

- **1.4.3 コントラスト(AA)**: 本文・ラベル・ボタン文字は背景比 4.5:1 以上 (テストで強制)
- **1.4.11 非テキストコントラスト**: ボタン枠・入力枠・フォーカスリングは 3:1 以上
- **1.4.1 色だけに頼らない**: 状態は色+文字/形で示す (チップにラベル文字あり、を維持)
- **2.4.7 フォーカス可視**: キーボードフォーカスが見えること
- **2.5.8 ターゲットサイズ**: タップ対象は 24px 以上 (できれば 44px)
- **1.4.4 テキスト拡大**: 200% ズームでレイアウトが壊れないこと
- **1.4.12 テキスト間隔**: 行間・字間を締めすぎない (行間 1.5 目安)

## このユーザー固有の要件 (最優先・妥協しない)

- **低コントラストが見分けられない**: 「薄いグレーの補足文字」は禁止。muted でも 4.5:1 以上
- **乱視**: ダークモード単独は目が痛い → **ライトモード必須** (`prefers-color-scheme` 両対応)。
  ライトの背景は純白でなくわずかにトーンを落とす (#f5f5f0 系)。ダークで彩度の高い光る色を広面積に使わない
- **斜視 (左右視差で行の段ずれ)**: 行の水平アンカーを明確に —
  行ごとの明確な境界線 or 縞 (zebra)、要素の揃え位置を統一、浮遊する小さなテキストを作らない

## 禁止事項

- スクショを見ずに UI 変更を出すこと
- ダークのみ/ライトのみの確認で済ませること
- コントラストテストを弱める方向の変更 (基準は上げてよいが下げない)
